# Lumina Note ユーザーガイド（日本語）

このガイドは 2 部構成です。前半は **どの機能がどこにあるか** の対応表。後半はその機能を実際につなげて使うためのスターターワークフロー。

## 機能の場所

本文中の名称：

- **左 Ribbon**：ウィンドウ左端に並ぶアイコン列。
- **コマンドパレット**：`Ctrl/Cmd + P`。
- **選択ツールバー**：エディタで文字を選ぶと出てくる小さなツールバー。

| やりたいこと | 場所 |
|---|---|
| コマンドパレットを開く | `Ctrl/Cmd + P`、または Ribbon 上部の ⌘ アイコン |
| Vault 全文検索 | Ribbon → 虫眼鏡アイコン（左サイドバーが検索パネルに切り替わる） |
| AI チャットを開く | Ribbon → Bot アイコン |
| ファイル一覧 / エディタ | Ribbon → ファイルアイコン |
| Vault 内の画像を一括管理 | Ribbon → 画像アイコン |
| ナレッジグラフ（Vault 全体） | Ribbon → ネットワークアイコン、またはコマンドパレットで "Show graph" |
| 現在のノートのローカルグラフ | ナレッジグラフと同じパネル。開いているノートに応じて自動で切り替わる |
| プラグインを管理 | Ribbon → パズルアイコン |
| 設定を開く | Ribbon 下部の歯車アイコン |
| ダーク / ライト切替 | Ribbon 下部の太陽 / 月アイコン |
| アプリ更新の確認 | Ribbon 下部のダウンロードアイコン |
| テーマを選ぶ / 自作する（公式 15 種＋カスタム） | 設定 → General |
| モデルと API Key の設定 | 設定 → AI |
| 同期（WebDAV / セルフホストアカウント / モバイル QR ペアリング） | 設定 → Sync |
| HTTP / SOCKS プロキシ | 設定 → Network |
| 音声録音をそのままノート化 | 左サイドバーのクイックアクション内のマイクボタン |
| PDF に注釈 | ファイルツリーで `.pdf` を開く（タブが PDF リーダーに切り替わる） |
| 選択テキストを AI へ / フラッシュカード生成 | エディタで文字を選択 → 浮かぶ選択ツールバー |
| `[[WikiLink]]` のホバープレビュー | wikilink にカーソルを乗せる（エディタ・閲覧モード・ファイルツリー・グラフで有効） |
| カスタム slash コマンドを登録 / 編集 | AI チャット入力で `/` → "管理" |
| エージェントスキルを実行（workspace / user / built-in） | コマンドパレットで "Open Skill Manager"、または AI 入力で `/skill` |
| 現在の AI 会話をエクスポート | チャット上部のツールバー |
| iOS / Android 端末をペアリング | 設定 → Sync → Mobile Gateway → 端末で QR を読む |
| クロスネットワーク同期 | `server/` をデプロイ（`docs/self-host.md`）→ 設定 → Sync で同じアカウントにログイン |

## 5 分セットアップ

1. [Releases](https://github.com/blueberrycongee/Lumina-Note/releases) から Lumina Note をインストールして起動。
2. ローカルフォルダを **Vault** として選択。
3. **設定 → AI** で API Key を入れてモデルを選ぶ（OpenAI / Claude / Gemini / DeepSeek / Moonshot / 智譜 / Groq / OpenRouter / Ollama / OpenAI 互換エンドポイントなど）。
4. 最初のノートを作る。`[[` と打つと既存ノートを補完候補として出す。Enter で新規作成も可能。
5. **ナレッジグラフ**（Ribbon ネットワークアイコン）を開き、2 つのノートが接続されていることを確認。
6. **AI チャット**（Bot アイコン）で、いま書いたノートについて何か質問する。

この 6 ステップが通れば、残りの機能はすべて同じ基本要素（ファイル＋AI＋グラフ＋同期）に対する別の UI 入口です。

## スターターワークフロー

### A. デイリーノート → 構造化された知識

1. デイリーノートに素早く書く。最初から構造を整える必要はない。
2. テーマが繰り返し出てきたら `[[WikiLinks]]` でつなげる。ホバープレビューで指している先が正しいか確認できる。
3. AI チャットでエージェントにアクション項目の抽出やセクション分割を依頼。プランを承認すると直接ファイルを書き換える。
4. ナレッジグラフ（Ribbon ネットワークアイコン）に切り替えて孤立ノートを探す。リンクし忘れたページが見つかる。

### B. PDF → 再利用可能な Markdown ノート

1. PDF を Vault に置き、ファイルツリーから開く。タブが PDF リーダーに切り替わる。
2. ハイライト、下線、注釈を付ける。終わったら注釈を Markdown として保存。
3. AI チャットでスコープを限定（例：「ハイライト部分のみを要約」）。
4. 自分の結論とタグを追記し、`[[WikiLinks]]` で知識ネットワークに織り込む。

### C. エージェントによるリファクタ

1. **先に**スコープを決める：「このファイル」または「`notes/research/` 以下すべて」。
2. プランを先に出させてから実行させる。エージェントランタイムは plan→apply に対応。
3. 段階的に進める。エージェントは普通の FS ツールでファイルに書くだけ。隠し動作はない。
4. 重要な見出しと書き換え部分は保存前に必ず人間がチェック。

### D. スマホへ同期

1. **同じ Wi-Fi**：**設定 → Sync → Mobile Gateway** を開いて QR を表示、モバイルアプリで読む。リレー不要。
2. **異なるネットワーク**：リレーサーバーをデプロイ（`docs/self-host.md`）→ デスクトップ（設定 → Sync）とモバイル両方で同じアカウントにログイン。

## プライバシーとデータ境界

- Vault はデフォルトでローカル保存。クラウドモデルや同期を有効にしない限りデータは端末から出ない。
- クラウドモデルへ送られるのはプロンプトに含めた内容のみ。エージェントが Vault 全体を勝手に読み出すことはない。
- 機密データには、別の Vault と別のプロバイダを組み合わせるか、Ollama でローカルモデルに統一するのが安全。

## 簡易 FAQ

### AI が応答しない

- API Key を確認（設定 → AI）。
- モデル ID がプロバイダの実際のモデル名と一致しているか確認。
- ネットワーク / プロキシ設定（設定 → Network）を確認。

### README に書いてある機能が見つからない

コマンドパレット（`Ctrl/Cmd + P`）で数文字入れて検索すると、Ribbon に出ていない機能の多くもヒットする。

### 最初に何を覚えるべき？

WikiLinks → AI チャット → ナレッジグラフ。この 3 点が手に馴染んだら、残りはショートカットの問題。
